# Code of Conduct

## Our Commitment

The FixMyText frontend is an open-source project built and maintained entirely online. Every interaction — pull requests, issues, code reviews, discussions, and comments — happens in digital spaces. We are committed to making these spaces welcoming, respectful, and productive for everyone, regardless of background, identity, or experience level.

## Expected Behavior

All participants in this project are expected to:

- **Be respectful in code reviews.** Critique the code, not the person. Frame feedback constructively — suggest improvements rather than just pointing out flaws. Remember that contributors may be submitting their first open-source PR.
- **Write clear, considerate communication.** Tone is hard to read online. Default to kindness. Use clear language, avoid sarcasm in technical discussions, and assume good intent.
- **Welcome newcomers.** Many contributors will be students, first-time open-source contributors, or developers learning React. Be patient with questions and guide people to the right resources (README, docs, existing examples).
- **Respect differing skill levels and perspectives.** Not everyone has the same experience with React, Redux, or frontend tooling. A question that seems basic to you may be genuine for someone else.
- **Stay on topic.** Keep issue threads and PR discussions focused on the technical matter at hand. Take unrelated conversations to appropriate channels.
- **Give credit.** Acknowledge others' contributions and ideas. If you build on someone's suggestion, mention it.
- **Follow project standards.** Adhere to the coding conventions, commit message format, and PR guidelines outlined in [CONTRIBUTING.md](CONTRIBUTING.md). Consistent code is respectful to future contributors who will read it.

## Unacceptable Behavior

The following will not be tolerated in any project space — issues, PRs, discussions, comments, commit messages, or any other communication channel:

- Insulting, demeaning, or dismissive comments directed at a person (e.g., "this code is terrible," "you clearly don't understand React")
- Harassment, threats, or intimidation of any kind
- Discrimination based on age, disability, ethnicity, gender identity, experience level, nationality, appearance, race, religion, sexual identity, or any other personal characteristic
- Publishing others' private information (email, real name, location) without explicit consent
- Sustained disruption of discussions, spamming issues or PRs, or deliberately wasting maintainers' time
- Sexualized language, imagery, or unwelcome advances
- Dismissing or attacking someone for asking a question or not knowing something

## Scope

This Code of Conduct applies to all online spaces associated with the FixMyText frontend project:

- GitHub issues, pull requests, code reviews, and discussions
- Commit messages and code comments
- Any linked communication channels (Discord, Slack, forums, etc.)
- Social media interactions when representing or discussing the project

## Maintainer Responsibilities

Project maintainers are responsible for:

- Enforcing this Code of Conduct consistently and fairly
- Responding to reports promptly and confidentially
- Removing, editing, or rejecting contributions (comments, code, issues) that violate this Code of Conduct
- Setting the tone by modeling respectful behavior in their own reviews and communications

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please report it to **conduct@velobits.org**. All reports will be reviewed and handled confidentially. You will not face retaliation for making a good-faith report.

When reporting, please include:

- A link to the comment, issue, PR, or discussion where the behavior occurred
- A description of what happened
- Any additional context that may be helpful

## Enforcement

Maintainers will respond to violations based on severity and pattern:

### 1. Reminder

**For:** First-time minor issues — an unintentionally harsh code review comment, a dismissive tone, or language that could be more welcoming.

**Action:** A private message pointing out the issue and asking the contributor to adjust their tone. No public action taken.

### 2. Warning

**For:** A clear violation or repeated minor issues after a reminder.

**Action:** A public or private warning. The contributor may be asked to edit or remove the offending content. Continued violations will result in escalation.

### 3. Temporary Restriction

**For:** Serious violations or a pattern of behavior that harms the community.

**Action:** Temporary loss of the ability to comment, submit PRs, or participate in discussions for a defined period. The contributor will be notified privately with the reason and duration.

### 4. Permanent Ban

**For:** Severe or repeated violations that demonstrate an unwillingness to follow community standards, including sustained harassment or discrimination.

**Action:** Permanent removal from all project spaces with no opportunity for appeal.
